# Messages 页面 Table 列配置功能实现计划

## 项目研究结论

通过分析 `Messages.vue` 文件，发现以下情况：

1. **当前状态**：
   - 页面已经集成了 SearchForm 组件，并传递了 `@column-settings` 事件监听器
   - 存在 `handleColumnSettings` 函数，但仅显示 "列设置功能开发中..." 的提示
   - 表格包含多个列：选择列、客户名称、手机号、到期时间、处理状态、授权起始日期、授权结束日期和操作列

2. **需要实现的功能**：
   - 创建列设置抽屉/弹窗
   - 实现列的显示/隐藏配置
   - 持久化列配置状态
   - 更新表格列的显示/隐藏

## 实施步骤

1. **定义列配置数据结构**
   - 创建表格列配置的数据源
   - 为每列定义唯一标识、显示名称和默认显示状态

2. **实现列设置抽屉**
   - 在模板中添加 ElDrawer 组件
   - 实现列设置的 UI 界面，使用 ElSwitch 组件控制每列的显示/隐藏

3. **实现列配置逻辑**
   - 添加列配置的响应式数据
   - 实现列配置的加载和保存逻辑
   - 实现列配置的本地存储持久化

4. **更新表格列显示**
   - 修改表格列的 v-if 条件，根据配置显示/隐藏列
   - 确保选择列和操作列始终显示（可选）

5. **实现列设置事件处理**
   - 更新 `handleColumnSettings` 函数，打开列设置抽屉
   - 实现列配置变更的处理函数

6. **测试和验证**
   - 测试列配置的显示/隐藏功能
   - 验证配置的持久化效果
   - 确保页面布局和功能正常

## 潜在依赖和考虑因素

- **依赖**：Element Plus 组件库（已使用）
- **存储**：使用 localStorage 存储列配置
- **兼容性**：确保与现有代码的兼容性
- **用户体验**：确保列配置操作简单直观

## 风险处理

- **风险**：列配置可能影响表格布局
  **应对**：确保至少保留核心列的显示

- **风险**：本地存储可能被清除
  **应对**：提供默认列配置作为回退

- **风险**：列配置与现有功能冲突
  **应对**：确保选择列和操作列始终可用

## 预期效果

- 用户可以通过 SearchForm 组件的列设置按钮打开列配置抽屉
- 用户可以通过开关控制表格列的显示/隐藏
- 配置会自动保存并在页面刷新后保持
- 表格会根据配置动态更新列的显示状态
